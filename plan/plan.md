## 🤖 AI 사용 전략 (토큰 절약 가이드)

### Opus 모델로 할 일 (이 파일에 기록)
- 새 기능의 설계/기획 결정
- 복잡한 버그의 근본 원인 분석  
- 아키텍처 변경이 필요한 큰 리팩토링
- 이 plan.md 업데이트

### Sonnet 모델로 할 일 (실제 코딩)
- 기획이 이미 정해진 기능 구현
- 단순 스타일/레이아웃 수정
- 파일 1~2개 수정으로 끝나는 작업
- 버그 픽스 (원인 파악 후)

### 세션 시작 방법
```
1. 이 plan.md를 먼저 읽어서 현황 파악
2. 오늘 할 작업을 "Phase X의 [ ] 항목 완료하기" 형태로 명시
3. 기획/설계가 필요하면 Opus → 구현은 Sonnet으로 전환
```
